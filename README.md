# SIEM Log Analysis with Splunk

## Objective
To analyze system logs using Splunk SIEM and detect suspicious activity.

## Tools Used
- Splunk Enterprise
- Sample log data

## Methodology
1. Installed and configured Splunk SIEM
2. Uploaded log files for analysis
3. Performed log searches using Splunk queries
4. Identified suspicious patterns in logs

## Key Findings
- Detected multiple failed login attempts
- Identified repeated patterns in log entries
- Observed potential brute-force indicators

## Skills Demonstrated
- SIEM usage (Splunk)
- Log analysis
- Threat detection
- Query writing (Splunk SPL)

## Conclusion
This project demonstrates practical experience using a SIEM tool to analyze logs and detect potential security threats.
